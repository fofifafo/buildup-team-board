# 보안 업그레이드 적용 가이드

이번 업그레이드로 바뀌는 것:
- 로그인이 **Firebase Authentication**(구글 정식 인증)으로 처리됩니다.
- 회원/게시글/지원서 데이터가 분리되고, **서버 수준 접근 권한**이 적용됩니다.
  - 회원 정보(전화번호, 생년월일 등): 본인과 관리자만 열람 가능
  - 지원서/연락: 해당 글 작성자와 관리자만 열람 가능
  - 게시글: 승인된 회원만 열람 가능
- 관리자는 별도 비밀번호 대신 **관리자 권한이 부여된 계정**으로 로그인합니다.

⚠️ 주의: 기존 데이터(이전 방식으로 가입한 회원, 기존 게시글)는 새 구조로 이전되지 않습니다.
지금은 테스트 단계라 데이터가 거의 없으니 새로 시작하는 것이 깔끔합니다.

---

## 1단계. Firebase 콘솔에서 Authentication 켜기

1. https://console.firebase.google.com → 프로젝트(meta-g-star) 선택
2. 왼쪽 메뉴 **빌드 > Authentication** 클릭 → **시작하기**
3. **로그인 방법(Sign-in method)** 탭 → **이메일/비밀번호** 클릭 → **사용 설정** 켜기 → 저장
   - (아이디 로그인은 내부적으로 이메일 형식으로 변환해서 처리하므로, 사용자는 그냥 아이디만 입력하면 됩니다)

## 2단계. Firestore 보안 규칙 교체

1. 왼쪽 메뉴 **빌드 > Firestore Database** → **규칙** 탭
2. 기존 내용을 전부 지우고, 이 프로젝트의 `firestore.rules` 파일 내용을 그대로 붙여넣기
3. **게시** 버튼 클릭 (꼭!)

## 3단계. 코드 교체 및 배포

1. `src/App.jsx` 와 `src/firebase.js` 를 새 파일로 덮어쓰기
2. 로컬 테스트: `npm run dev`
3. 배포:
   ```
   git add .
   git commit -m "Firebase Auth 및 보안 규칙 적용"
   git push
   ```

## 4단계. 관리자 계정 만들기 (최초 1회)

1. 배포된 사이트(또는 로컬)에서 **일반 회원가입**으로 관리자용 계정을 하나 만듭니다.
   (예: 아이디 `admin`, 본인 정보 입력)
2. Firebase 콘솔 → **Firestore Database** → `users` 컬렉션을 엽니다.
3. 방금 가입한 계정의 문서를 클릭합니다. (userId 필드로 구분 가능)
4. **필드 추가** 버튼으로 아래 두 가지를 수정/추가합니다:
   - `status` 필드 값을 `pending` → `approved` 로 변경
   - 새 필드 추가: 이름 `isAdmin`, 유형 `boolean`, 값 `true`
5. 이제 그 계정으로 사이트에 로그인하면 헤더에 "관리자" 메뉴가 보입니다.
6. 이후 다른 회원들의 가입 승인/거절/삭제는 사이트 안의 관리자 페이지에서 처리하면 됩니다.

## 알아둘 점

- **회원 탈퇴/삭제의 한계**: 관리자가 계정을 삭제하면 그 회원의 프로필 정보가 삭제되고 사이트 이용이 즉시 차단됩니다. 다만 Firebase 인증 계정 자체(아이디/비밀번호 껍데기)는 남아있어서, 같은 아이디로 재가입은 안 됩니다. 완전히 지우려면 Firebase 콘솔 → Authentication → 사용자 탭에서 해당 계정을 수동 삭제하면 됩니다.
- **비밀번호 재설정**: 아이디가 실제 이메일이 아니라서 "비밀번호 찾기" 이메일 발송은 불가능합니다. 회원이 비밀번호를 잊으면 관리자가 Authentication 탭에서 해당 계정을 삭제하고 재가입하도록 안내하세요.
- **개인정보 보호 수준**: 이제 데이터베이스에 직접 접근해도 본인 인증 없이는 회원 정보를 읽을 수 없습니다. 다만 여전히 학생 개인정보(전화번호, 생년월일)를 다루는 서비스이므로, 수집 목적을 안내하고 필요 최소한으로 운영하시길 권합니다.
